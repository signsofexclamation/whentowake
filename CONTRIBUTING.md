# contributing

thanks for your interest in contributing to *whentowake*.

## what is welcome

- bug fixes
- performance improvements
- code quality improvements
- translations

## what is not welcome

- feature additions without prior discussion
- redesigns or UI changes without prior discussion

if you have an idea, open an issue first. if it makes sense, we can talk 
about it before you spend time writing code.

## how to contribute

1. fork the repository
2. create a branch for your change
3. make your changes
4. open a pull request with a clear description of what you changed and why

## code style

keep it consistent with the existing code. no new dependencies without discussion.

## issues

if you find a bug, open an issue with enough detail to reproduce it.
